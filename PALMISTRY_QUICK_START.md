# Быстрый старт: Система Вариаций Хиромантии

## ✅ Что было сделано

Расширена система предсказаний хиромантии с 3 вариаций на **30 вариаций** для каждой области жизни (любовь, карьера, судьба).

**Результат:**
- ❌ Раньше: семья видит одинаковые предсказания
- ✅ Теперь: каждый скан ладони дает уникальный текст

---

## 📦 Новые файлы

### 1. `lib/palmistry-readings.js` (⭐ Главный файл)

Содержит:
- `LOVE_READINGS` — 30 вариаций о любви
- `CAREER_READINGS` — 30 вариаций о карьере
- `DESTINY_READINGS` — 30 вариаций о судьбе

Каждая вариация использует Барнум-эффект + психологию.

### 2. `lib/palmistry-reading-mixer.js` (Опционально)

Дополнительный модуль для **компонентного подхода**:
- Разбивает текст на компоненты (введение, инсайт, завершение)
- Позволяет комбинировать их для 810+ вариаций
- Используется, если нужна еще большая вариативность

### 3. Этот файл + `PALMISTRY_VARIATIONS_GUIDE.md`

Документация и примеры.

## 🚀 Как использовать

### Текущая система (уже внедрена)

Больше ничего менять не нужно. `palmistry-generator.js` автоматически:

```javascript
// Когда пользователь сканирует ладонь:
const reading = generateReading({
  sessionId: 'unique-session-123', // каждый скан = новый sessionId
  name: 'Анна',
  gender: 'female',
  focusArea: 'love'
});

// reading.preview → 1 из тысяч комбинаций
// reading.full → 1 из 30 вариаций о любви (или карьере/судьбе)
```

**Пользователь видит:**
- Разный текст каждый раз сканирует ладонь
- Если два человека сканируют в один день — они видят разные предсказания
- Текст звучит личным, но универсально релевантным

#### Вариант A: Еще больше вариативности (компонентный подход)

```javascript
const mixer = require('./lib/palmistry-reading-mixer');

// Результат: текст собран из компонентов (3 × 3 × 3 = 27 комбинаций)
```

#### Вариант B: Пользовательское сидирование (опционально)

Если хотите, чтобы один пользователь всегда видел одинаковую вариацию:

```javascript
const crypto = require('crypto');

function getReadingByUser(userId) {
  const hash = crypto.createHash('sha256').update(userId).digest('hex');
  const index = parseInt(hash.slice(0, 8), 16) % LOVE_READINGS.length;
  return LOVE_READINGS[index];
}

// Юзер1 всегда видит вариацию #7
// Юзер2 всегда видит вариацию #24
// Но соседи не видят одинаковых текстов
```

## ❓ FAQ

**Q: Все ли текст теперь случайный?**
A: Нет, по-прежнему используется seeded random по sessionId. Это значит:
- Если пользователь откроет одну сессию дважды → видит одинаковый текст (консистентность)
- Если открывает новую сессию → видит другой текст (вариативность)

**Q: Влияет ли это на производительность?**
A: Нет. Это просто выбор из массива — O(1) операция.

**Q: Нужно ли обновлять базу данных?**
A: Нет. Это чисто логика генерации на стороне сервера.

**Q: Можно ли запретить повторение одной вариации?**
A: Да, если добавить логику отслеживания (sessionStorage в фронтенде). Но обычно не нужно.
